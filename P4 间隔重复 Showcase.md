# P4 间隔重复 Showcase

> 这篇笔记自带几张闪卡，用来测试 FSRS 复习流程。
> 打开后，状态栏会出现 **"🎓 N 待复习"** 红色徽章——点它开始复习。

## 基础卡

Rust 的所有权三规则： #card
- 每个值有唯一所有者；离开作用域值被丢弃；同一时刻仅一个所有者。

为什么 Rust 不需要 GC？ #card
- 编译期通过所有权系统追踪值的使用，超出作用域自动 drop。

## 双向卡

Bidirectional link = 双向链接 #card-reverse
- 反向链接：其他笔记引用了我，我能反向看到它们

## 填空卡

Rust 用 {{c1::所有权}} 系统管理内存，无需 GC。 #cloze
Rust 借用规则：同一时刻只能有 {{c2::一个可变引用}}，或多个不可变引用。 #cloze

## 怎么玩

1. 保存这篇笔记后，状态栏右下角应该出现 `🎓 N 待复习` 徽章
2. 点它（或者按 `Ctrl+R`）打开复习面板
3. 看到问题 → 按 `Space` 翻面 → 用 `1/2/3/4` 评分：
   - **1 Again** — 完全忘了，~10 分钟后回来
   - **2 Hard** — 答对了但很费力
   - **3 Good** — 正常答对
   - **4 Easy** — 完全没问题
4. 评分后立即看到这条卡的"下次复习时间"

## FSRS 算法

Rust 端实现了 **FSRS v4** 算法（已通过集成测试）。参数：
- 默认目标保留率 90%
- Difficulty ∈ [1, 10]
- Stability 从 0.4 到 ~100 天动态调整
- 每次 Again → 难度 +2，稳定性按当前难度重算

## 数据存在哪

所有 FSRS 状态存在 `vault/.notevault/index.sqlite` 的 `cards` 表里：
- 重新打开 vault 自动恢复复习队列
- 删除笔记自动级联删卡
- 改笔记内容（不动 #card 标记）→ 卡的 ID 不变，复习进度保留
